# **Report on Identified Issues and Solutions**

## **1. Overview**

During the development and testing of the Smart Restaurant Ordering System, several critical issues were identified related to image rendering, project documentation, and local deployment. These issues impact usability, maintainability, and development workflow. This report provides a detailed explanation of each problem along with appropriate solutions.

---

# **2. Issue 1: Images Not Displaying in Customer and Admin View**

## **Problem Description:**

Even after adding images while creating menu items, the images are not visible in:

* Customer interface
* Admin dashboard

---

## **Root Cause:**

The issue typically arises due to one or more of the following reasons:

* Backend is not serving image files properly
* Incorrect image URL stored in database
* Frontend is not using the correct path to display images

---

## **Proposed Solution:**

### **1. Serve Static Files from Backend**

The backend must expose the folder where images are stored.

### **Implementation:**

```js
app.use("/uploads", express.static("uploads"));
```

---

### **2. Store Correct Image Path**

When saving image:

```js
image_url = "/uploads/" + filename
```

---

### **3. Correct Frontend Rendering**

Instead of directly using image path:

```js
<img src={item.image_url} />
```

Use:

```js
<img src={`http://localhost:5000${item.image_url}`} />
```

---

## **Expected Outcome:**

* Images display correctly in all views
* Improved UI/UX

# **4. Issue 3: Running Project Locally and Database Setup**

## **Problem Description:**

The user needs to:

* Run the project locally on their system
* Set up and connect the database

---

## **Proposed Solution:**

---

## **4.1 Running the Project Locally**

### **Step 1: Install Required Software**

* Node.js
* npm (Node Package Manager)

---

### **Step 2: Install Dependencies**

Run in both frontend and backend folders:

```bash
npm install
```

---

### **Step 3: Start Backend Server**

```bash
npm run dev
```

Server runs on:

```
http://localhost:5000
```

---

### **Step 4: Start Frontend**

```bash
npm start
```

Runs on:

```
http://localhost:3000
```

---

## **4.2 Database Setup (MongoDB)**

### **Option 1: Local MongoDB**

#### Install MongoDB and run:

```
mongodb://localhost:27017/restaurant
```

---

### **Option 2: MongoDB Atlas (Cloud)**

* Create cluster
* Get connection string
* Add to `.env` file

```env
MONGO_URI=your_connection_string
```

---

### **Backend Connection Example:**

```js
mongoose.connect(process.env.MONGO_URI)
```

---
