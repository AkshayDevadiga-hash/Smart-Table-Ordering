## 1. ❌ Review System Issues (Critical)

### Problems:

* Input fields clear automatically while typing
* Review not stored in database
* Reviews not visible on admin page

### Fix:

* Ensure form inputs (rating + comment) are **properly controlled**

  * Do not reset state on each keystroke
* On submit:

  * Call `POST /api/reviews` with:

    * `order_id`
    * `rating`
    * `comment`
* Validate:

  * Rating (1–5 required)
  * Comment ≤ 500 characters
* After success:

  * Show confirmation message
  * Disable resubmission

### Admin:

* Ensure `/admin/reviews`:

  * Calls `GET /api/reviews`
  * Displays rating, comment, date correctly
* Fix any field mismatch (`order_id` vs `orderId`)

## 3. ❌ Cancel Order Logic Bug (High Priority)

### Problem:

* Users can cancel after payment
* Admin sees invalid state: `cancelled + paid`
* No refund handling

### Fix Rules:

#### Do NOT allow cancel if:

* `payment_status = paid`
* OR `status = preparing | ready | delivered | completed`

### Frontend:

* Hide or disable “Cancel Order” button in these cases

### Backend:

* Add validation in `PATCH /api/orders/:id`:

  ```
  if (order.payment_status === "paid") {
    return error "Cannot cancel after payment"
  }
  ```
* Ensure consistent order state
