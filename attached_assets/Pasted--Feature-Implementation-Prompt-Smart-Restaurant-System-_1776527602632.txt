### 🔧 Feature Implementation Prompt – Smart Restaurant System

Enhance the existing Smart Restaurant Ordering System by implementing the following features and improvements:

---

## 1. ⭐ Customer Reviews System (UI + Flow)

### Customer Side:

* After order status becomes **"delivered" OR payment is completed**, show a **review form**
* Fields:

  * Rating (1–5 stars, required)
  * Comment (optional, max 500 chars)
* Submit via:

  * `POST /api/reviews`
* Ensure:

  * Only **one review per order**
  * Show success message after submission
* Redirect user back to menu or show “Thank you” screen

### Admin Side:

* Create a new page: `/admin/reviews`
* Fetch reviews using:

  * `GET /api/reviews`
* Display:

  * Rating ⭐
  * Comment
  * Date
* Add filters:

  * By rating (1–5)
  * By date range

---

## 2. 📄 About Page (Customer UI)

* Create `/about` page
* Add link/button in menu page (footer or header)
* Include:

  * Project/restaurant description
  * Key features (QR ordering, live tracking, etc.)
  * Contact info (optional)

---

## 3. ❌ Cancel Order (Customer Side)

* Add **“Cancel Order” button** on `/order/:orderId` page
* Only show button when:

  * Status = `pending` OR `received`
* On click:

  * Show confirmation popup: “Are you sure you want to cancel?”
  * Call:

    * `PATCH /api/orders/:id` → update status to `cancelled`
* After cancel:

  * Show message: “Order cancelled successfully”
  * Disable further actions
* Ensure:

  * Cancel is NOT allowed after `preparing` stage

---

* Example steps:

  * Received → Preparing → Ready → Delivered
* Highlight current step
* Update dynamically based on polling
