# SYSTEM ISSUE REPORT (AI-READABLE)

## CONTEXT

Smart Restaurant Ordering System has multiple logic, state, and calculation issues affecting billing, order persistence, and profit tracking.

---

## ISSUE 1: GST & TOTAL RETURNS NaN

**Problem:**

* GST and total amount evaluate to `NaN`

**Cause:**

* Undefined/null values in price/quantity
* Missing numeric conversion
* Invalid arithmetic operations

**Fix Requirements:**

* Ensure all numeric inputs are parsed (`Number()` or equivalent)
* Add validation before calculation
* Handle null/undefined safely

---

## ISSUE 2: NO RUNNING TOTAL (CART NOT MAINTAINED)

**Problem:**

* System does not maintain cumulative order total
* Each item handled independently

**Cause:**

* No cart array or persistent order structure

**Fix Requirements:**

* Maintain `cart[]` (array of items)
* Recalculate total using all items in cart
* Display subtotal, GST, and final total

---

## ISSUE 3: ORDER DATA OVERWRITTEN

**Problem:**

* Previous orders are lost when new item is added

**Cause:**

* Order stored as single object instead of collection

**Fix Requirements:**

* Use array-based storage (`orders[]` or `cart[]`)
* Append instead of overwrite

---

## ISSUE 4: ORDER NOT PERSISTED UNTIL PAYMENT

**Problem:**

* System forgets previous items before payment
* Only last item retained

**Cause:**

* No persistent storage or session tracking

**Fix Requirements:**

* Store full order until payment is completed
* Use database/local storage/session

---

## ISSUE 5: NO PAYMENT FUNCTIONALITY

**Problem:**

* No payment option on customer side

**Cause:**

* Missing payment flow implementation

**Fix Requirements:**

* Add `Pay` action
* Introduce payment state tracking

---

## ISSUE 6: PROFIT UPDATED BEFORE PAYMENT (CRITICAL)

**Problem:**

* Admin profit increases even if customer has not paid

**Cause:**

* Profit linked to order creation instead of payment confirmation

**Fix Requirements:**

* Introduce `paymentStatus`
* Only include orders where:

  ```
  paymentStatus == "paid"
  ```

---

## ISSUE 7: NO SEPARATION OF STATES

**Problem:**

* Order lifecycle and payment lifecycle are mixed

**Fix Requirements:**
Define two independent fields:

```
orderStatus: pending | preparing | delivered
paymentStatus: pending | paid
```

---

## ISSUE 8: NO TOTAL TRACKING ON CUSTOMER PAGE

**Problem:**

* Customer cannot see full bill across multiple items

**Fix Requirements:**

* Display:

  * Item list
  * Subtotal
  * GST
  * Final total

---

## ISSUE 9: CURRENCY FORMAT NOT STANDARDIZED

**Problem:**

* Amounts not displayed in rupee format

**Fix Requirements:**

* Format all amounts as:

  ```
  ₹<amount>.00
  ```

Order {

1. Total = sum(price × quantity for all items)
2. GST = total × 0.18
3. Final Amount = total + GST
4. Orders must persist until paymentStatus == "paid"
5. Profit = sum(totalAmount where paymentStatus == "paid")
6. Do NOT overwrite existing order data
7. Cart must accumulate items until payment
