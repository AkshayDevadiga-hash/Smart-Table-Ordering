# ⚠️ What Needs Improvement (Important)

## 🚨 1. Frontend `server.js` (Big Issue)

```bash
frontend/server.js  # Node static server + proxy
```

👉 This defeats your “separation” goal.

### Why this is a problem:

* You’re mixing backend logic into frontend again
* Vercel **won’t support this properly**
* Adds unnecessary complexity

---

### ✅ Fix:

👉 Remove this completely:

```bash
frontend/server.js ❌
```

Instead:

* Use:

  * **Vercel / Netlify** → for frontend
* Call backend directly using:

```env
API_URL=https://your-backend.onrender.com
```

---

## 🚨 2. Monorepo (pnpm workspace)

Right now:

```bash
pnpm-workspace.yaml
```

👉 This is fine for dev, BUT:

### Problem:

* Makes deployment harder
* Overkill for your use case

---

### ✅ Recommendation:

If you want simplicity:

👉 Split into 2 repos:

```bash
restaurant-backend/
restaurant-frontend/
```

OR keep monorepo but deploy separately

---

## 🚨 3. API Proxy (Not Needed)

You wrote:

```bash
frontend proxies /api/* → backend
```

👉 This is unnecessary if you already use `API_URL`

---

### ✅ Better Approach

Frontend calls directly:

```js
fetch(`${API_URL}/orders`)
```

No proxy needed ❌

---

## 🚨 4. Backend Port Confusion

You wrote:

```bash
Frontend default: 3000
Backend default: 5000
But API_URL default: http://localhost:8080 ❌
```

👉 This inconsistency will break things

---

### ✅ Fix:

Keep it consistent:

```env
# backend
PORT=5000

# frontend
API_URL=http://localhost:5000
```
